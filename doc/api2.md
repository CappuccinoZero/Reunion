## 1. 用户关注
### 路径: user/follow
### 内容：关注其他用户，以便优先获得其他用户发表的内容
### 请求参数：
uId: 关注者用户 id

targetUId: 被关注者用户 id

follow：Boolean : 0 表示关注 1 表示取消关注

### 响应

code：
* 200 成功
* 400 目标用户uid不存在

msg：信息

因为这个原子性操作有时会发生错误
如果follow = true(关注目标用户)，此时服务器记录的也是true(已经建立关注)，那么直接返回200:成功 就可以了
以此类推
